# Output Contracts

GroundLine output should be structured enough to hand to another skill, LLM, or
agent.

## GroundLine Assessment

```text
GroundLine Assessment:
- current conclusion:
- verified state:
- capability gaps:
- side-effect boundary:
- recommended mode:
- recommended next skill or tool:
- exact next prompt:
- verification checklist:
```

## GroundLine Radar

```text
GroundLine Radar:
- changed sources:
- new sources:
- removed sources:
- potential impact:
- research packet:
- upgrade task candidates:
- verification checklist:
```

## Boundary

```text
Boundary:
- classification:
- approval needed:
- intended side effect:
- read-only checks:
- execution allowed:
- secret value printed: false
```
