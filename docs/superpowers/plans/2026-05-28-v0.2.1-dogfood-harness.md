# v0.2.1 Dogfood Harness Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and release v0.2.1 by closing the v0.2.0 dogfood gap with a reproducible provider dogfood harness.

**Architecture:** Add one stdlib-only harness that stages the package into a temporary provider home, verifies provider manifests and skill counts, probes runtimes when requested, and checks the shared dogfood scenario contracts. Keep the harness read-only for real provider homes and record the result in docs.

**Tech Stack:** Python stdlib, unittest, GitHub CLI release flow, GroundLine skill index and output contracts.

### Task 2: Harness Implementation

**Files:**
- Create: `scripts/groundline_dogfood.py`
- Modify: `scripts/validate_pack.py`

- [x] **Step 1: Implement provider and scenario inventory**

The script defines:

```text
providers: codex, claude_code, antigravity
scenarios: long-context-handoff, completion-proof, expansion-control
expected skills: package-agent-task, close-live-work, stabilize-release-cut
```

- [x] **Step 2: Implement staged package checks**

With `--stage-package`, the harness creates a temporary home, copies provider manifests and `skills/` into provider target paths, and verifies target manifest and skill count.

- [x] **Step 3: Implement optional runtime probes**

With `--probe-runtimes`, the harness runs `codex --version`, `claude --version`, and `agy --version`, or uses `--bin-dir` for deterministic tests.
